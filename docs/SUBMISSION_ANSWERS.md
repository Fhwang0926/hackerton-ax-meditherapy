# MediInsight 예선 질문 5문항 답변 초안

## 1. 무엇을, 누가, 어떤 상황에서 쓰나요?

MediInsight는 메디테라피 마케팅·브랜드 실무자가 제품별 공개 고객 경험을 조사하고 인스타그램 콘텐츠 초안을 만들 때 사용하는 Codex 플러그인입니다. 사용자는 최초 한 번 자사몰과 외부 판매·리뷰 채널 URL을 등록합니다. 이후에는 제품 이름만 지정하면 저장된 URL을 다시 사용합니다. 플러그인은 공개 페이지와 렌더링된 공개 리뷰 iframe을 수집하여 근거 보고서, 4컷 인스타그램 이미지 3장, 제품 홍보 이미지 1장, 캡션, 법률 예비 검수 결과를 생성합니다.

## 2. 왜 이 문제를 선택했나요?

메디테라피 제품에 대한 고객 경험은 자사몰과 외부 리뷰 채널에 나뉘어 있습니다. 공식 PDRN 제품 페이지는 공개 구조화 데이터와 리뷰 위젯에서 리뷰 수와 고객 후기를 제공하고, 화해에도 별도 제품 페이지가 존재합니다. 채널을 따로 보면 사용 기간, 피부 타입, 사용 상황, 불만과 재구매 신호를 한 번에 비교하기 어렵습니다. 따라서 “리뷰가 없다”가 아니라 “공개된 고객 경험이 채널별로 분산되어 근거 기반 콘텐츠로 전환하기 어렵다”를 해결 대상으로 정했습니다.

- 메디테라피 공식 제품 페이지: https://meditherapy.co.kr/product/detail.html?cate_no=118&display_group=1&product_no=2039
- 화해 제품 페이지: https://www.hwahae.co.kr/products/2166543

## 3. 플러그인은 어떻게 작동하나요?

플러그인은 저장된 URL을 Chrome으로 렌더링하고 HTML과 화면 캡처를 보존합니다. 메디테라피 공식몰의 공개 Crema `list_v3` 리뷰 iframe은 기본 2페이지까지 추가로 렌더링하여 리뷰 본문, 별점, 후기 이미지 URL을 수집합니다. URL이 같은 수동·자동 자료의 채널명을 통합하고, 정규화된 본문 지문으로 중복을 제거합니다. Evidence Engine은 사용 기간, 피부 타입, 사용 마찰, 루틴과 재구매 표현을 추출하여 모든 인사이트에 evidence id를 연결합니다. 이어서 보고서와 인스타그램 이미지를 만들고, 로컬 Law MCP로 이미지 제목·본문·CTA·제품 광고 문구를 검수합니다. 위험 문구는 수정하고 위험 문구가 이미지 자체에 포함되면 해당 원본 이미지를 격리합니다. 마지막으로 법률 인용 URL의 접근 상태와 전체 실행 단계를 파일로 남깁니다.

## 4. AI를 어떻게 활용했나요?

Codex는 여러 URL을 방문하고 렌더링 결과를 파일로 보존하는 실행 오케스트레이터로 사용했습니다. 텍스트 분석에서는 리뷰의 시간축과 사용 맥락을 구조화하고, 근거 id가 연결된 콘텐츠 초안을 생성합니다. VLM은 후기 사진으로 효능이나 피부 상태를 판정하지 않고, 화면에 보이는 문구 전사와 제품 배치·레이아웃 확인에만 제한합니다. 생성 문구는 Law MCP의 화장품 광고 위험 규칙을 통과한 수정본으로 다시 렌더링합니다. API 키가 필요한 외부 생성 서비스는 사용하지 않습니다.

## 5. 어떻게 검증했나요?

Python 단위 테스트로 광고 위험 표현, MCP JSON-RPC 호출, 리뷰 지표 계산, URL 기억과 병합, 동적 Crema 리뷰 파싱, 중복 제거, 위험 이미지 격리를 검증했습니다. 실제 메디테라피 PDRN 페이지를 사용한 통합 실행에서는 공개 리뷰 16건을 확보했으며 그중 10건은 렌더링된 리뷰 iframe에서 자동 수집했습니다. 후기 이미지 URL 25개가 근거에 연결됐고, 2개 실제 채널을 포함한 데이터 품질 상태가 `ready`로 계산됐습니다. 보고서, 법률 결과, 1080×1080 PNG 4개가 생성되는지 확인했고, 원본 이미지의 `100% 환불보장` 문구가 탐지되면 최종 광고에서 해당 자산이 제외되는 것도 확인했습니다. 공식 인용 URL 접근 결과는 `source_verification.json`, 실행 결과는 `run_manifest.json`으로 남겨 재검증할 수 있습니다.

## 제출 전 갱신 항목

- 최종 실행 시점의 리뷰 수와 테스트 수로 숫자를 다시 맞춥니다.
- 원본 AI 대화 로그를 편집하지 않고 `logs/`에 추가합니다.
- 법률 검수는 법률 자문이 아닌 예비 QA임을 유지합니다.
