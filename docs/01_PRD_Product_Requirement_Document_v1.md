# 01. Product Requirement Document (PRD)

Version: 1.0

## 1. 문서 목적(Document Purpose)

### 1.1 목적
본 문서는 MediInsight 프로젝트의 기능 요구사항, 시스템 요구사항, 개발 범위 및 구현 기준을 정의한다.

이 문서를 기준으로 Codex는 다음을 구현할 수 있어야 한다.

- 프로젝트 구조 생성
- Plugin 개발
- Skill 작성
- Crawler 구현
- Report Generator 구현
- VLM 분석
- Law MCP 연동

본 문서는 개발의 Single Source of Truth로 사용한다.

---

## 1.2 프로젝트 개요

**프로젝트명**
MediInsight

**한 줄 설명**
공개된 화장품 판매 채널의 고객 데이터를 수집하여 근거 기반 인사이트를 생성하고, 즉시 활용 가능한 마케팅 콘텐츠를 생성한 후 Law MCP를 이용해 광고 문구까지 검증하는 Codex Plugin.

---

## 1.3 프로젝트가 해결하는 문제

메디테라피 제품은 자사몰, 올리브영, 네이버, 쿠팡 등 여러 채널에서 판매된다.

리뷰, 이미지, 평점, 사용 기간, 피부 타입 등의 데이터가 여러 채널에 분산되어 있기 때문에 전체 고객 경험을 하나의 관점에서 분석하기 어렵다.

또한 리뷰를 콘텐츠 자산으로 재활용하는 과정과 광고 문구 법률 검토 과정이 대부분 수작업으로 이루어진다.

---

## 1.4 현재 업무 방식

1. 쇼핑몰 방문
2. 리뷰 수집
3. 엑셀 정리
4. 키워드 분석
5. 콘텐츠 기획
6. 디자인
7. 법률 검토
8. 게시

---

## 1.5 문제점

### 문제 1
고객 데이터가 여러 채널에 분산되어 있다.

### 문제 2
고객 Journey를 시간축으로 분석하기 어렵다.

### 문제 3
리뷰를 마케팅 자산으로 활용하지 못한다.

### 문제 4
콘텐츠 제작 시간이 오래 걸린다.

### 문제 5
법률 검토가 마지막 단계에서 이루어진다.

---

## 1.6 프로젝트 목표

- 공개 데이터 자동 수집
- 데이터 통합
- Insight 생성
- Report 생성
- Carousel 생성
- Product Ad 생성
- Law MCP 검수
- 최종 Export

---

## 1.7 핵심 가치

Evidence → Insight → Content → Compliance

콘텐츠 생성이 목적이 아니라 근거 기반 의사결정을 자동화하는 것이 목적이다.

---

## 1.8 왜 Codex Plugin인가?

- 다수 URL 자동 방문
- 크롤링
- 스크린샷
- OCR/VLM
- Report 생성
- 파일 생성
- Law MCP 호출
- 자동 워크플로우 실행

일반 챗봇이 아니라 자동화 Agent에 적합한 프로젝트이다.
